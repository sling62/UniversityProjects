Block Level Tags

Block level tags provide basic content structure for a web page. This includes Headings, Paragraphs, Horizontal rules, List organisation etc.

Headings

Paragraphs, breaks and rules

Paragraph tags are used as a basic container for body text. Line breaks and horizontal rule tags are used to seperate text.

The reason for using a line break over a paragraph is if you dont want to have a full paragraph break and just want to go onto the next line, a line break does not replace a paragraph tag for containing text. So if a br tag is used, it should be used within a paragraph tag and its semantic meaning for seperation not for appearance purposes.

Lists

HTML supports 3 types of list structure:

  1. Unordered(bullet points)
  2. Ordered(numbered points)
  3. Definition(descriptive terms)

The focus of lists is the semantics/organisation not visual display(lists can be fine-tuned using css).Well structured lists form the basis of many dynamic menu and tab effects(with the aid of css and javascript).

Unordered List

An unordered list is a bulleted list. You can style a unordered list bullet points by using the <type> attribute for the <ul> tag. By default it is a disc. However you can also use square and circle as well.

  • A bullet list starts with a ul tag
  • Each item is surrounded by li tags
  • The list ends with a ul tag

Ordered List

This list is used to create an indexed list, such as a numbered or alphabetical list.

  1. A bullet list starts with a ol tag
  2. Each item is surrounded by li tags
  3. The list ends with a ol tag

HTML5: Flow Elements

Flow elements was introduced in order to reflect a logical structure for "modern" web documents.

Some flow elements at block-level include:

  • <article>
  • <section>
  • <nav>
  • <aside>
  • <header>
  • <footer>

<article>

This tag is a container(normally situated in a <section> tag) in a document, page, application, or site.It is usually accompanied with a footer and header like with any article. Examples of where using a article tag is appropriate include: a forum post, a magazine or newspaper article, or a blog entry.

<section>

This tag represents an independent section of functionality contained within an HTML document. It is usually accompanied with a heading. Be careful not to confuse the <section> tag as a general container; if you want a general container which is contained within other container's then a <div> should be used, for instance if you are sectioning a part of a document for styling purposes, use a div instead. A good way to think about is that a section should be independent and the contents can be seperate from everything else in a document.

<nav>

This tag is as the name sugguests, the navigation bar section of the page.Inside it provides links to other pages or to other parts within the page. Not all links of a document must be in a <nav> tag, which is intended only for major block of navigation links; so for instance the <footer> tag usually has a block of links that does not need to be in a <nav> tag.

<aside>

This tag represents a section of a document that contains content which accompanies the main content of the document, like a sidebar. The <aside> tag should not be used for parenthesized text(text within bracket) because it is considered as part of the main flow.

<header>

This tag is normally used for introductory content or content of which helps navigate the page. This could include heading tags, a logo, a search form etc. The <header>,<footer> and <article> tag is not sectioning content therefore is usually contained within the <section> tag as show below.

<footer>

This tag is as the name sugguests a container for content usually at the bottom of a section tag. A footer normally contains information about the author of the section or page, copyright data or links to related sites.The author content is usually put in an <address> tag which is placed in the <footer> tag.

Summary: Block Level Tags

In HTML, tags are either 'inline' or block-level elements. Block-level elements exist in their own virtual 'box' and are always followed by a carriage return (like hitting the 'enter' key after typing in some text). An example of a block tag is a <p> tag (paragraph tag). Testing out these on a HTML document, you will see what happens to your page when you use multiple <p> tags and highlight the background of these tags(using CSS).
