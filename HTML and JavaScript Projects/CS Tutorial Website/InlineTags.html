Inline Tags

Inline Tags are tags which are nested within a Block level tag.There are two types of Inline tags; Logical and Physical. An inline tag does not start on a new line and only takes up as much width as necessary. Examples of Inline tags are: <span>, <img> and <a>.

Introduction to Logical and Physical tags (what is the difference?)

Logical tags

Physical tags

Logical tags represent the structure and meaning of a document, with only suggested renderings for their appearance which could vary depending on different browser's system configurations.

The physical tags represent the visual effects on specific words in a document that are intended to only be used when suitable like highlighting a keyword, and do not have any reason to it's semantic meaning.

The original implementation of HTML contained almost entirely of logical tags, The reason for this was that HTML was viewed as more of a structural language rather than a visual language.

As HTML was developed by browser-makers like Netscape and Microsoft, a large number of physical tags were added, with very little development of the logical tag set.

The HTML 4.0 standard and now HTML 5.0, attempts to remedy the over-use of Physical tags by introducing some new logical tags and declaring some of the physical tags and attributes as "deprecated" in favor of using CSS stylesheets instead.

Logical tags

Logical tags are designed to describe (to the browser) the enclosed text's meaning or semantic meaning. By placing text in between the tags below you are telling the browser that the text has some greater importance.

For instance by default all browsers make the text appear italic when in between the <em> and </em> tags, but the point to take away from this is that the <em> tag implies the importance of that text. This is of a great importance to search engines like google which look for these tags to help figure out what the page is about.

  • <em>
  • <strong>
  • <cite>
  • <code>
  • <abbr> and <acronym>
<em>Empharsise Text</em>

The <em> tag mainly stresses emphasis of its contents, while the <i> tag represents text that is like normal text but is more specific, such as the name of a movie or book, a foreign word, or when the text refers to the definition of a word.

An example for <em> could be: "Just do it already!", or "We had to do something about it".

A person or software reading the text would pronounce the words in italics with an emphasis.

<strong>Make text strong</strong>

Similar to the <em> tag, <Strong> is normally percieved as just conveying a stronger emphasis, in HTML5, the tag is described as representing "strong importance for its contents". There is a big difference between emphasis and importance.

While Emphasis is used to change the meaning of a sentence ("I love carrots" vs "I love carrots"), Strong is used to indicate bits of a sentence added importance (e.g. "Warning! This is very dangerous").

Both Strong and Emphasis can be nested to increase the relative degree of importance or stress emphasis, respectively.

<cite>Cite it right</cite>

Unlike the <i> tag, the <cite> tag specifically is used as a reference to a creative work(e.g. A painting). the tag requires that when you assign the URL or title of that work. This could be in an abbreviated form following the correct citation format. An example is:

<p><cite>The Scream</cite> by Edward Munch. Painted in 1893.</p>

<code>Code it</code>

The <code> tag is used in a HTML document to represent computer code. By default, it is displayed in a browser as monospace font.

For example declaring a Javascript variable in HTML:

<code> var x = 0; </code>

Tip: This tag is not deprecated, but it is possible to achieve better effect with CSS.

<abbr>WWW.</abbr>

The HTML Acronym Element (<acronym>) as the name sugguests allows clearly to indicate a sequence of characters that make up an acronym for a word. This element has been removed in HTML5. So use <abbr> element. The HTML <abbr> element represents an abbreviation of a word and gives the option to supply a description for it. If included, the title attribute must contain this full description and nothing else.

An example of using an abbreviation tag in HTML: <abbr title = "History"> Hist.</abbr>

Physical tags

Physical tags on the other hand provide specific instructions on how to display the text they enclose. Physical Inline tags just provide visual feedback, however many of these tags are now deprecated, so it is best to use logical tags.

  • <b>
  • <i>
  • <big>
  • <small>
  • <sub>
  • <sup>
  • <u>
  • <s>
<b>Bold Text</b>

In HTML the <b> tag is used to indicate a piece of text of which is different visually from normal text, without showing any particular importance or relevance.It is normally rendered in a boldface font.

Examples of which the <b> tag is used for are: keywords, product names in a article, or other pieces of text which the normal presentation would be in a bold font.

For Example: <p>this text has been bolded to <b>stand out</b> visually from the rest of the text</p>

Tip: If the text has no semantic meaning at all on using the <b> tag, then the css property font-weight with bold value would be a more suitable alternative for making the text bold.

For Example: <p style="font-weight:bold;">Yay, I'm Bold!</p>

<i>Italicize Text</i>

In HTML the <i> tag is used to indicate a piece of text that has a more specific meaning than to the other normal text.Examples of this are: technical terms, foreign language phrases, or fictional character thoughts. It is normally presented in italic font type.

An example of using this tag: <p> The latin word <i class = "latin">aurum </i> translates to gold in English.</p>

Tip:you can use CSS styling in order to produce italic font using "font-style: italic;"

<big>Make Text Bigger</big>

In HTML Big tag (<big>) is used to make the text font size one size bigger (ie. from small to medium, or from large to x-large) until it reaches the browser's maximum font size.

For Example in HTML: <p>This is a normal sentence. <big>

This whole sentence is in bigger letters.

</big></p>

An alternative is in CSS:</p>This is a normal sentence in a paragraph. <span style="font-size:1.2em"> This part of the sentence is in bigger letters.</span></p>

<small>Make Text small</small>

In HTML the small tag (<small>) makes the text font size one size smaller (ie. from large to medium, or from small to x-small) until it reaches the browser's minimum font size. In HTML5, this tag was given a new purpose in order to represent side-comments and small print(like copyright and legal text) so that it does not need to be styled to be this way.

For Example in HTML:<p> This is a normal sentence. <small>This sentence is in smaller letters.</small></p>

Alternative in CSS:</p> This is a normal sentence. <span style="font-size:0.8em"> This sentence is in smaller letters.</span></p>

<sub>written in subScript</sub>

In HTML the <sub> tag indicates a piece of text that should be presented differently, for typographic reasons lower and usually smaller, than other normal pieces of text. This tag is mainly only used for typographical reasons; like in mathematical and scientific formulas, changing the position of the text so that it accurately represents its meaning.

An example of this: <p>The chemical formula of water: H<sub>2</sub>O</p>

Tip:This tag main purpose is not for appearance. If this type of styling is wanted to be used for appearance then CSS should be used ie. the vertical-align property with the sub value will achieve the same effect.

<sup>superScripted!</sub>

In HTML the <sup> tag similar to the <subScript> tag indicates a piece of text that should be presented differently for typographic reasons, however higher and smaller instead of lower and smaller, than normal text. This tag is mainly only used for typographical reasons; like in mathematical and scientific formulas, changing the position of the text so that it accurately represents its meaning.

An example of this: <p>x<sup>2</sup> = 25 </p>

Tip:This tag main purpose is not for appearance. If this type of styling is wanted to be used for appearance then CSS should be used ie. the vertical-align property with the sub value will achieve the same effect.

<u>Importent to underline titles</u>

In HTML the <u> tag creates text which is underlined(a line under the base of the text).

An example of this: <p><u>Important:</u> see <u>Rules and Regulations</u> section for details</p>

Like majority of physical tags, the <u> tag was deprecated in HTML 4 and XHTML 1, but came back in HTML5 with more semantic meaning.

Tip:If you want to underline text for appearance then you should use a <span> tag, or another semantically appropriate element and style it with the CSS text-decoration property, with the underline value.

Tip: Avoid using the <u> element where it could be confused for a hyperlink.

<s>cross out unassecasry words</s>

In HTML the <s> tag creates text which have a strikethrough, or a line through it.The the <s> tag is mainly used to indicate text that is no longer relevant or no longer accurate.

Tip: The <s> tag is not appropriate when indicating document edits; for that, use the <del> and <ins> tags, as appropriate.

HTML Example:<p><s>Price Before: £14</s> Price After: £10 </p>

CSS Example:<p><span style="text-decoration:line-through;">Price Before: £14 <span> Price After: £10</p>

HTML5: Phrasing Elements

In HTML, a phrase tag is used to indicate that a block of text has structural meaning. For example, in logical tags using the <abbr> tag indicates that the word or phrase contained within is an abbreviation or acronym(see Logical tags section).

It was Introduced to reflect the logical structure for "modern" web documents, it reduces the need for extra common id and class attributes.

Summary: Inline tags

In HTML, tags are either 'inline' or block-level elements. Inline tags (elements) become a part of the 'flow' of text in which they are inserted and have no virtual 'box' around them and don't have the carriage return either(space after you hit enter). You can have either Logical(focuses on semantic meaning) or Physical(focuses on visual meaning) inline tags. An example of a Physical tag is the <b> (bold) tag. It is important to know the difference between them and when it is appropriate to use them or CSS. Try the tags out on various browsers and you'll see the difference to your page when you use <b> tags and <strong> tags.
